# CampusConnect Chatbot - Setup and Run Instructions

## STEP 1: Install Dependencies

Open terminal in: F:\CampusConnect\frontend\src\pages\Chatbot

# Activate virtual environment (if not already activated)
.\venv\Scripts\activate

# Install all required packages
pip install -r requirements.txt

# This will install:
# - sentence-transformers (for embeddings and re-ranking)
# - scikit-learn (for similarity calculations)
# - rank-bm25 (for hybrid search)
# - All other existing dependencies


## STEP 2: Verify Your Data Files

Make sure your PDF files are in:
F:\CampusConnect\frontend\src\pages\Chatbot\data\

The system now supports ALL these file types:
- PDF files (*.pdf)
- Text files (*.txt)
- CSV files (*.csv)
- Excel files (*.xlsx)
- Word documents (*.docx)
- JSON files (*.json)


## STEP 3: Run the Chatbot Server

# Make sure you're in the Chatbot directory with venv activated
cd F:\CampusConnect\frontend\src\pages\Chatbot
.\venv\Scripts\activate

# Run the Flask server
python app.py

# You should see:
# - "Loading cross-encoder model..."
# - "Cross-encoder loaded successfully"
# - "RAG Search initialized with X documents"
# - Server running on http://localhost:5000


## STEP 4: Test the Chatbot

Open your browser and go to:
http://localhost:5000

You should see the API info page.


## STEP 5: Run Your Frontend

Open a NEW terminal and run:
cd F:\CampusConnect\frontend
npm run dev

Then access the chatbot page in your application.

## TROUBLESHOOTING:

If you see errors:
1. Make sure venv is activated
2. Install dependencies: pip install -r requirements.txt
3. Check GROQ_API_KEY is set in .env file
4. Verify PDF files are in the data/ folder
5. Delete data/vector_store/ folder and restart to rebuild index


## PERFORMANCE NOTES:

- First query takes longer (loads models ~5-10 seconds)
- Subsequent queries are fast (1-3 seconds)
- Re-ranking adds ~0.5-1 second per query
- The accuracy improvement is worth the slight delay!


## REBUILDING THE INDEX:

If you add new PDFs:
1. Stop the server (Ctrl+C)
2. Delete the vector_store folder: rmdir /s data\vector_store
3. Restart: python app.py
4. The system will rebuild with all PDFs
